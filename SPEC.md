# קטלוג ספרים, חוברות ודברי דפוס — מסמך אפיון

מסמך זה מרכז את האפיון של אפליקציית הקטלוג. הוא הבסיס לפיתוח, וניתן לעריכה לפני כתיבת קוד.

**גרסה:** 0.1 (טיוטת אפיון) · **תאריך:** 23.07.2026

---

## 1. מטרה

אפליקציית ווב לקטלוג אוסף ספרים, חוברות ודברי דפוס — כללי ותורני — עם דגש ראשוני על אוסף קטן של חוברות וספרים נדירים לצורכי מחקר. המבנה מודולרי מראש כדי להכיל בהמשך את כל הספרייה.

## 2. עקרונות מנחים

- **מודולריות:** אוסף חדש = שורה בטבלה, לא מקרה מיוחד בקוד.
- **מסד נתונים = Google Sheets:** נראה, ניתן לעריכה ידנית, מגובה בענן, בלי תשתית לתחזק.
- **קוד בגיט:** ניהול גרסאות מלא.
- **תמונות ב-Google Drive:** בעת הצורך.
- **אחידות נתונים:** ערכים קבועים (נושאים, מדפים, מו״לים, אוספים) נשלטים דרך רשימות נפתחות מלשוניות-עזר, לא הקלדה חופשית.

## 3. מחסנית טכנולוגית

ארכיטקטורה סטטית — ללא שרת משלנו וללא חשבון שירות. ראה [SETUP-GITHUB-PAGES.md](SETUP-GITHUB-PAGES.md).

| רכיב | בחירה |
|---|---|
| Frontend | אתר סטטי (HTML/CSS/JS), RTL בעברית, מתארח ב-**GitHub Pages** |
| "Backend" | **Google Apps Script** (Web App) צמוד לגיליון — קורא/כותב ומחזיק סודות |
| מסד נתונים | Google Sheets |
| אחסון תמונות | Google Drive (שלב מאוחר) |
| חילוץ טקסט משער | Claude Vision API (המפתח נשמר ב-Apps Script, שלב 2) |
| השלמת ISBN | Google Books (ציבורי, ללא מפתח) |

> **למה לא Next.js/חשבון שירות?** GitHub Pages מארח אתר סטטי בלבד — אין איפה להסתיר מפתח סודי.
> לכן ה"שרת" הוא Apps Script אצל גוגל, שרץ בהרשאות הבעלים ומחזיק סודות בצד השרת.

### 4.2 לשוניות-עזר (Lookup)

- **`נושאים`** — רשימה היררכית. כל שורה = נתיב מלא, למשל `תורני / חסידות / חב״ד`, `כללי / היסטוריה / יהדות מזרח`.
- **`מדפים`** — רשימת מיקומים פיזיים.
- **`מו״לים`** — רשימת מוציאים לאור.
- **`אוספים`** — רשימת האוספים.
- **`הגדרות`** — קידומות מספור קטלוגי וערכי ברירת מחדל.

## 5. שיטת המספור הקטלוגי

מבנה: `<קידומת>-<מספר רץ 4 ספרות>`

- קידומת לפי אזור-על: `ת` = תורני, `כ` = כללי. (ניתן להוסיף/לערוך בלשונית `הגדרות`.)
- המספר הרץ מתקדם אוטומטית בנפרד לכל קידומת: `ת-0001`, `ת-0002`, `כ-0001`...
- המספר **קבוע** — אינו משתנה גם אם הפריט עובר מדף או נושא.

## 6. צירי סיווג (עצמאיים)

לכל פריט ארבעה צירי סינון בלתי-תלויים: **מספר קטלוגי · מדף · נושא · מו״ל** (בנוסף לאוסף ולתגיות).

## 7. דרכי הזנת פריט

1. **ידני** — טופס מלא. ברירת המחדל, המדויקת ביותר לספרים נדירים.
2. **צילום שער → חילוץ טקסט** — צילום השער, Claude Vision ממלא שדות אוטומטית, המשתמש מאשר/מתקן.
3. **חיפוש ISBN** — השלמה אוטומטית לספרים מודרניים.

## 8. תוכנית בנייה בשלבים

- **שלב 0 — שלד:** ריפו גיט, אתר סטטי RTL, Apps Script עם סכימת הגיליון. ✅
- **שלב 1 — CRUD ידני:** הוספה/עריכה/מחיקה/חיפוש/סינון, מספור קטלוגי אוטומטי, לשוניות-עזר מתעדכנות. ✅ *(כאן ניתן כבר לקטלג את האוסף הנדיר.)*
- **שלב 2 — צילום שער → חילוץ אוטומטי** (Claude Vision דרך Apps Script).
- **שלב 3 — השלמת ISBN** (Google Books).
- **שלב 4 — ליטוש:** העלאת תמונות ל-Drive, ייצוא ביבליוגרפי, סינון מתקדם, גיבוי.
